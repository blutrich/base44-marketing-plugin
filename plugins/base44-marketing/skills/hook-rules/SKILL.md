# Hook Creation Rules

Rules for creating engaging, on-brand hooks that avoid AI-detection patterns.

## What We DON'T Use

| Banned Pattern | Why | Example |
|----------------|-----|---------|
| Contrarian hooks | We don't need to be "against" something to stand out | "Stop doing X..." |
| FOMO tactics | No fear-based urgency | "Don't miss out", "Before it's too late" |
| Negative framing | We lead with possibility, not problems | "You're doing it wrong" |
| Arrow bullets (>, ➡️, ▸) | Creates a cold, listicle feel - AI detection flag | "> Feature one" |
| "We're excited to announce" | Corporate speak | Any variation |
| TV-ad tagline cadence | Sounds like a commercial, not a builder | "One workspace. Unlimited builders. No friction." |
| Advertising melody | Billboard/radio spot rhythm | "Build faster. Ship smarter. Scale infinitely." |
| "Nobody tells you" | Fake drama, AI pattern | "Nobody's talking about this but..." |
| "Let's dive in" | YouTube intro energy | "Let's break this down", "Let's explore" |
| "No X, no Y, just Z" | AI contrast framing | "No plugins, no exports, just..." |
| Self-narration | Announcing your own points | "Here's why this matters", "The kicker?" |
| Rhetorical transitions | Fake suspense | "The catch?", "The brutal truth?", "But here's the thing" |
| Em dashes | Biggest AI tell in 2026 | Use commas, periods, or parentheses instead |

## What We DO Use

### Formatting Rules
- Emojis only for bullet points and emphasis
- Clean, warm visual style
- Positive, builder-focused energy
- No arrows - use emoji bullets instead

### 5 Approved Hook Styles

#### 1. Result-First
Lead with the outcome, then explain.

```
$350K saved.
One app.
Built in a weekend.
```

#### 2. Builder Spotlight
Feature a real person's achievement.

```
Sarah launched her SaaS yesterday.
No code. No investors. Just Base44.
```

#### 3. Possibility Hook
Ask "what if" to spark imagination.

```
What if your next idea was live by Friday?
```

#### 4. Social Proof
Show momentum through numbers.

```
12 apps launched this week.
Here's what they built:
```

#### 5. Direct Value
Benefit-focused statements grounded in real outcomes. NOT TV-ad cadence.

```
We just cut build time from 2 weeks to 3 hours. Here's how.
```

#### 6. Hormozi Formula (for landing page H1s)
**[Specific Result] + [Surprising Context] + [Implied "how"]**

The result must be real (from Slack/metrics). The context attacks the competitor wedge. The "how" is never stated, it creates curiosity that pulls the reader in.

```
His AI agent was live in 2 minutes. No terminal. No Docker. No API keys.
  Result: AI agent was live
  Context: 2 minutes (vs hours for competitors)
  Implied how: what tool makes this possible?

She replaced 4 SaaS tools with one app she built over lunch.
  Result: replaced 4 tools
  Context: built over lunch
  Implied how: how is that even possible?

700 people asked for early access in one day. The feature isn't even public yet.
  Result: 700 signups
  Context: one day, not even public
  Implied how: what is it?
```

**When to use:** Landing page H1s, hero sections, campaign headlines. Not for social posts (use styles 1-5 for those).

**Validation:** The result must trace to a real proof point. If you can't cite the source, the hook is invented and gets rejected.

## Approved Emoji Bullets

Use these instead of arrows:

| Emoji | Use Case |
|-------|----------|
| ✅ | Completed items, features included |
| 🚀 | Launches, speed, momentum |
| 💡 | Ideas, insights, tips |
| 🔥 | Hot takes, exciting news |
| ⚡ | Speed, instant results |
| 🎯 | Goals, accuracy, targeting |
| 💪 | Empowerment, capability |
| 🛠️ | Building, tools, creation |

## Examples

### ❌ Wrong (Contrarian + FOMO + Arrows)
```
Stop wasting time on code that doesn't matter.
> Your competitors are already shipping faster
> Don't get left behind
> The window is closing
```

### ✅ Right (Positive + Emoji + Builder-focused)
```
Ship your idea this week.

🛠️ Visual builder for everything
⚡ Deploy in one click
🚀 Real customers by Friday

Your turn.
```

## Hook Validation Checklist

Before using any hook, verify:

1. [ ] No arrow bullets (> ➡️ ▸)
2. [ ] No FOMO language ("don't miss", "before it's too late", "left behind")
3. [ ] No negative framing ("stop wasting", "you're doing it wrong")
4. [ ] No contrarian angle for its own sake
5. [ ] No TV-ad cadence (stacked declarative fragments)
6. [ ] No advertising melody (billboard/radio spot rhythm)
7. [ ] No em dashes (use commas, periods, or parentheses)
8. [ ] No "let's" openers ("let's dive in", "let's explore")
9. [ ] No self-narration ("here's why this matters", "the kicker?")
10. [ ] No "No X, no Y, just Z" contrast framing
11. [ ] No rule-of-three (two items or four+, never three)
12. [ ] Leads with outcome, not problem
13. [ ] Uses approved emoji set for bullets (max 1-2)
14. [ ] Sounds like a builder talking to a builder (Maor Test)

## Integration with Agents

When linkedin-specialist, x-specialist, or copywriter generates hooks:

```
1. Generate hook using approved styles
2. Check against banned patterns list
3. Validate no arrow bullets present
4. Ensure positive framing
5. Use approved emoji set for bullets
6. Lead with outcomes, not problems
```

## Channel-Specific Hook Rules

### LinkedIn
- Hook must stop the scroll in 2 seconds
- First line under 150 characters
- Question hooks work well for engagement
- Numbers in first line boost CTR

### X (Twitter)
- Hook IS the tweet for single posts
- Thread hooks: intrigue without clickbait
- "The results were honestly shocking" style works
- Numbered threads (1/) signal more coming

### Email
- Subject line = hook (under 50 chars)
- Preview text extends the hook (90 chars)
- Curiosity + benefit combo works best
