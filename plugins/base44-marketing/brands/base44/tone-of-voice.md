# Base44 — Tone of Voice Guide

---

## What This Document Is

This is the definitive guide to how Base44 sounds. Every piece of content, every post, every email, every announcement should pass through this filter. It's grounded in how Maor actually writes on LinkedIn and X, refined by team feedback across 38+ content iterations.

If something in this guide conflicts with your instinct to "make it more polished," trust the guide. The lack of polish IS the voice.

---

## The Voice in One Sentence

A founder who shipped something real today and is telling you about it the way he'd tell a friend, with specific numbers and zero preamble.

---

## The Maor Test

Before publishing anything, ask one question:

**Would Maor copy-paste this into his LinkedIn and hit post without editing?**

- If yes: ship it.
- If you'd need to "clean it up" or "make it more professional" first: you've already failed. His voice IS the polish.

## Voice Character

### The "Cool Big Brother"

We talk at eye level. Supportive, honest, occasionally teasing. You want to be our friend and trust our voice.

| This | Not This |
|------|----------|
| Peer-to-peer | Corporate-to-consumer |
| Celebrate builder wins | Lecture about features |
| Share honestly, including failures | Only show highlights |
| Make builders feel capable | Overwhelm with complexity |

### Four Pillars

| Pillar | What It Means |
|--------|---------------|
| **Builder-first** | We talk to people who make things. No corporate distance. No talking down. |
| **Ship-obsessed** | Only post when you shipped. The shipping IS the content. |
| **Specific** | Real numbers, real names, real timelines, real apps. |
| **Unfiltered** | Casual, honest, occasionally surprised by your own success. |

### Patterns You'll Notice

Study the real posts above. Here's what's consistent:

1. **He opens with the thing.** No preamble, no context-setting, no "I've been thinking about..."
2. **He uses "I" more than "we."** It's personal, founder-voiced, not corporate.
3. **He uses dashes (-) for lists, not bullet points or arrows.**
4. **He uses asterisks for emphasis:** "$100K *profit*" not "$100K profit" or "**$100K profit**"
5. **He uses `:)` smileys, never emoji** (with rare exceptions on X when being very casual).
6. **He uses "1/" numbering for X threads**, never "Thread:" or "🧵 Thread"
7. **His longest posts are feature roundups** with dash-lists.
8. **He occasionally ends with a question** but only after a substantive feature roundup: "What else do you want to see?" (Never as a standalone hook.)
9. **He says "I honestly think" for opinion posts**, not "Here's the thing" or "Hot take."
10. **He quotes himself.** "This aged well" with a quote-tweet of an old post is peak Maor.

## Vocabulary

### Words We Use

| Word | Instead Of | Why |
|------|-----------|-----|
| builders | users, customers | Our identity |
| ship / shipped | deploy, release, launch | Builder vocabulary |
| drop / dropped | announce, introduce | Casual energy |
| go live | launch | Less corporate |
| build | develop, create, craft | Direct |
| vibe coding | no-code | Our category name |
| batteries included | full-stack, all-in-one | Maor's signature |

### Intensifiers (Use Sparingly)

| Word | Context |
|------|---------|
| "just" | Simplicity or recency: "Just shipped," "Just dropped" |
| "wild" | Surprising results: "Wild that this took 3 hours" |
| "crazy stuff" | Genuine excitement |
| "finally" | Long-awaited features |
| "in case you blinked" | Rapid shipping |

### Words We Never Use

| Never | Use Instead |
|-------|------------|
| users / customers | builders |
| deploy / launch | ship / go live |
| leverage / utilize | use |
| facilitate | help |
| streamline | simplify |
| craft (figurative) | write, make |
| empower | let, enable |
| showcase | show |
| enhance | improve |
| innovative / groundbreaking / cutting-edge | (say what's actually new) |
| seamless / robust / transformative | (say what it actually does) |
| game-changing / unprecedented | (show the result with a number) |

## Post Structures

### Feature Announcement

```
[What shipped, no preamble]                 <- 1 sentence, 5-15 words

[What it enables for builders]              <- 1-2 sentences
[Specific proof: number or timeline]        <- 1 sentence

[Demo video or screenshot]
```

**Example:**
> Your Base44 app can now send and receive emails.
>
> Gmail integration is live. Connect your account, test in sandbox mode, go live when you're ready.
>
> Took us 3 days.
>
> [demo video]

### Milestone

```
[The number. Period.]

[Context: what it means, 1-2 sentences]
[Credit: community, team, or timing]
```

**Example:**
> $1M ARR.
>
> 6 months ago this was a side project I built during reserve duty. 400K+ builders later, here we are.
>
> Still processing.

### Builder Spotlight

```
[Name] built [app] on Base44. [One stat.]

[What the app does, 1 sentence.]
[How they built it, with timeframe.]
```

**Example:**
> Sarah built a $50K ARR business on Base44 in 3 months.
>
> Started as a weekend project. Described what she wanted, shipped v1 on Monday, had paying customers by Friday.

### Build-in-Public

```
[Honest statement]

[Context or explanation, 1-2 sentences]
[What it means]
```

**Example:**
> 90% of my code was written by Claude.
>
> I'm not a senior developer. I described what I wanted and iterated on the result.
>
> The skill isn't coding anymore. It's knowing what to build.

## The Anti-Patterns

These are the most common mistakes. Each was flagged in real content reviews.

### 1. TV-Commercial Cadence

"One workspace. Unlimited builders. No friction."
"Build faster. Ship smarter. Scale infinitely."

**Why it fails:** This rhythm belongs on a billboard, not in a LinkedIn post. Read it aloud. If it sounds like a voiceover for a 30-second spot, rewrite it.

**Fix:** Write a full sentence. "We shipped workspaces. Your whole team can build in the same project now."

### 2. Self-Narration

"Here's why this matters:" / "The key takeaway is..." / "Now for the interesting part:"

**Why it fails:** You're announcing what you're about to say instead of saying it.

**Fix:** Delete the narration line. Start with the actual point.

### 3. Significance Inflation

"A testament to what's possible..." / "Marking a pivotal moment..." / "Setting the stage for..."

**Why it fails:** You're telling me something is important instead of showing me why.

**Fix:** Replace with a number. "$350K Salesforce contract replaced by one Base44 app" beats "a testament to the power of vibe coding."

### 4. Generic Hooks

"Just dropped a new feature." / "Another update just shipped."

**Why it fails:** You could swap any feature into these hooks. The hook should only make sense for THIS specific feature.

**Fix:** "Your app can now send emails" only works for email integration. That's specific enough.

### 5. Rule-of-Three

"Build, ship, and scale." / "Fast, flexible, and free."

**Why it fails:** Trios are the most common AI writing pattern. Cut to two or expand to four+. Two is almost always enough.

### 6. Em Dashes

"Base44 — the platform that — lets you build — anything."

**Why it fails:** Em dashes are the biggest AI tell in 2026. Zero tolerance.

**Fix:** Use commas, periods, or parentheses.

### 7. Contrast Framing

"It's not about code. It's about ideas." / "No plugins. No exports. Just results."

**Why it fails:** Just say what it IS. Skip the theatrical setup.

**Fix:** "Describe what you want. Base44 builds it."

### 8. Engagement Bait

"What would you build?" / "Agree or disagree?" / "Drop a 🔥 if..."

**Why it fails:** Maor never does this. Neither do we.

**Fix:** End naturally or don't end with a question at all.

## Channel Rules

### LinkedIn

- **Length:** 150-300 words. Shorter is better.
- **Structure:** Short paragraphs (1-3 sentences). Line breaks between ideas.
- **Media:** Demo video preferred. Almost never post without media.
- **Hashtags:** Never.
- **Emojis:** Never in body text. A `:)` is fine.
- **Links:** Never in main post (kills reach). First comment if needed.
- **CTAs:** Natural or none. "Try it" is acceptable. "What would you build?" is banned.

### X / Twitter

- **Length:** Single tweets under 280 characters. Threads 3-12 tweets.
- **Structure:** One idea per tweet. Number with "1/", "2/", etc.
- **Tone:** More raw than LinkedIn. More personal, more casual.
- **Hashtags:** Never (except tagging @Base44 brand account).
- **Emojis:** Almost never. Occasional 🧵 for thread indicator. Very rare casual emoji only on team celebration tweets.
- **Threads:** Start with "1/" and a hook. Use 🧵 emoji to signal thread. Financial or milestone threads get the most engagement.
- **Quote tweets:** Short, genuine reactions. "This aged well." Not promotional framing.
- **Emphasis:** Use asterisks for *emphasis* on key numbers: "$100K *profit*"

### Email

- **Length:** 150-200 words max.
- **Opening:** Start with the reader's interest, never with "I."
- **Format:** Include links to live URLs and repos. Show, don't describe.

### Discord / Community

- **Tone:** Most casual. Friends hanging out.
- **Emojis:** Acceptable here.
- **Engagement:** React to builder projects. Ask specific questions about their apps.

## Five Tests Before Publishing

1. **The Maor Test:** Would Maor post this on his LinkedIn exactly as written?
2. **The Swap Test:** Could you swap the feature name and the post still works? If yes, the hook is too generic.
3. **The Aloud Test:** Read it aloud. Does it sound like a TV commercial? Rewrite.
4. **The Number Test:** Is there at least one specific number? Add one.
5. **The Delete Test:** Can you delete the first sentence and the post still works? Then your real opening is sentence two.
